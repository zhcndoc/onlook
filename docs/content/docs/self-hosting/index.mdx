# 自托管 Onlook

Onlook 为需要完全控制其可视化开发环境的组织提供灵活的自托管选项。请选择最适合您基础设施和需求的部署方式。

## 部署选项

### 单机部署（推荐）
使用独立构建版本直接在单台服务器或虚拟机上部署 Onlook。这是自托管 Onlook 最简单高效的方式。

**适合：**
- 小型到中型团队（5-50 用户）
- 简单的生产环境部署
- 预算有限的组织
- 希望无需容器即可直接控制的团队

[查看单机部署指南 →](/self-hosting/single-machine)

### Docker Compose
使用 Docker 容器运行 Onlook，实现隔离部署。适合容器化环境。

**适合：**
- 已使用 Docker 的组织
- 开发和测试环境
- 需要容器隔离的团队
- 复杂的多服务部署

[查看 Docker Compose 指南 →](/self-hosting/docker-compose)

### 云端部署（企业版）
将 Onlook 部署到云基础设施，支持恰当的扩展、监控和企业功能。

**适合：**
- 大型团队与企业级部署
- 需要高可用性的组织
- 需要高级监控和合规性的团队
- 多区域部署

[查看云端部署指南 →](/self-hosting/cloud-deployment)

## 架构概览

Onlook 由以下组件组成：

- **Web 客户端**：Next.js 应用
- **后端**：使用 Supabase 提供数据库、认证和存储服务
