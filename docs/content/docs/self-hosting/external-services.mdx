# 外部服务

Onlook 需要多个外部服务来提供其功能。这些服务均可自托管，并采用 Apache 2.0 或同等宽松的许可协议。

## 自托管外部服务

如需自托管这些服务的协助，请通过 [founders@onlook.com](mailto:founders@onlook.com) 联系我们。

### 1. Supabase
- 对于可扩展部署，请按照官方的 [Supabase 自托管指南](https://supabase.com/docs/guides/self-hosting/docker) 使用 Docker 进行部署。
- 对于单机部署，您可以通过 CLI 运行 supabase，参考 [单机指南](/self-hosting/single-machine#3-start-backend-services)。

### 2. 沙箱提供者
- 对于可扩展部署，请使用他们的 [自托管指南](https://github.com/e2b-dev/infra/blob/main/self-host.md) 将 E2B 部署到 GCP。
- 对于单机部署，我们未来将添加一个完全本地的沙箱提供者。

### 3. AI 提供者
配置自定义 AI 提供者步骤：
1. 更新 [`packages/ai/src/chat/providers.ts`](https://github.com/onlook-dev/onlook/blob/main/packages/ai/src/chat/providers.ts) 中的提供者。我们已支持 Anthropic 和 OpenRouter 作为示例。您可以按照相同格式添加新的提供者。
2. 搜索并更新调用处，关键词为 [`initModel`](https://github.com/search?q=repo%3Aonlook-dev%2Fonlook+%22await+initModel%22&type=code)。
3. 在 `apps/web/client/.env` 文件中更新您的 API 密钥为该提供者所需的密钥。

注意：我们支持来自 [AI SDK 提供者](https://ai-sdk.dev/providers/ai-sdk-providers) 的任何提供者。您可以通过以下 AI SDK 提供者指南添加自定义提供者：[兼容 OpenAI](https://ai-sdk.dev/providers/openai-compatible-providers/custom-providers) 和 [社区](https://ai-sdk.dev/providers/community-providers/custom-providers)。
